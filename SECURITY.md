# Security Policy

## Supported Versions

The following table lists which versions of `node-tesseract-ocr` currently receive security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | ✅                  |
| < 1.0.0 | ❌                  |

Security fixes are applied to the latest minor/patch releases of the current major version only.

## Reporting a Vulnerability

If you discover a security vulnerability in `node-tesseract-ocr`, please report it responsibly and avoid public disclosure until it has been triaged and fixed.

### How to report

Please use one of the following options:

1. **GitHub Security Advisory**
   - Go to the repository on GitHub.
   - Open the **Security** tab and use **“Report a vulnerability”**.
   - This will create a private security advisory thread with the maintainers.

2. **Private contact**
   - If the Security tab is not available, open a **new issue** marked clearly as “Security” and **do not include exploit details**.
   - Ask for a private contact channel (e.g. email) to share the full report.

### What to include

To help triage and fix the issue efficiently, please include:

- A clear description of the vulnerability and its impact.
- Steps to reproduce (code samples, configuration, test data).
- The version(s) of `node-tesseract-ocr` you tested.
- Information about your environment:
  - Node.js version
  - Operating system and version
  - Installed Tesseract version
- Any suggested fixes or mitigations, if you have them.

### Our process

- Your report will be reviewed as soon as possible.
- If the issue is confirmed as a vulnerability:
  - A fix will be developed and tested.
  - A new release of `node-tesseract-ocr` will be published.
  - A short security note or advisory will be added to the release notes and/or GitHub Security Advisories.
- You will be informed about the status (accepted, needs more info, not applicable, etc.) through the same channel you used to report.

### Responsible disclosure

Please:

- Do not publicly disclose details of the vulnerability before a fix has been released.
- Avoid testing on production systems that you do not own or have explicit permission to test.
- Follow applicable laws and regulations while researching and reporting vulnerabilities.

Thank you for helping to keep `node-tesseract-ocr` and its users secure.
